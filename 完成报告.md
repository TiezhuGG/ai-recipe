## ⚠️ 关于AI功能的重要说明

### 你可能会遇到的情况

当你测试AI对话时，如果看到这样的回答：

```
这是一个很好的问题！建议您：
1. 多观察食材的变化
2. 掌握基本的烹饪原理
3. 多实践多总结
4. 可以参考专业烹饪书籍或视频教程
```

**这是备用答案**，说明豆包API调用失败了。

### 为什么会这样？

可能的原因：
1. 豆包API调用失败
2. API Key无效或过期
3. 网络连接问题
4. API配额用完

### 如何解决？

#### 步骤1: 运行测试脚本
```bash
cd backend
python test_cooking_ai.py
```

这会显示详细的错误信息。

#### 步骤2: 检查配置
打开 `backend/.env` 文件，确认：
```env
LLM_API_KEY=8b12a07e-b47a-4e39-8e36-c96c7ccced86
LLM_API_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
MODEL_NAME=doubao-seed-1-8-251228
```

#### 步骤3: 查看后端日志
启动后端时，注意看日志输出：
```
INFO: 调用豆包API回答烹饪问题 (尝试 1/3)
DEBUG: API URL: https://ark.cn-beijing.volces.com/api/v3/chat/completions
INFO: API响应状态码: 200
```

如果看到错误状态码（如401、403、500），说明API调用有问题。

#### 步骤4: 验证API Key
- 登录豆包API控制台
- 检查API Key是否有效
- 确认是否有足够的配额
