### 3. 浏览器测试

访问: `http://localhost:5173`

#### 测试清单

##### ✅ AI对话模式
1. 点击导航栏"去学厨房"
2. 默认进入"AI对话"模式
3. 输入问题："如何让肉更嫩？"
4. 点击"发送"或按Enter
5. 等待AI回答（应该看到"思考中..."动画）
6. 检查回答是否详细且相关
7. 尝试快速提问按钮
8. 检查对话历史是否正确显示

**成功标志**:
- 回答详细且相关（不是备用答案）
- 对话历史正确显示
- 时间戳正确

##### ✅ 技巧知识库模式
1. 点击"技巧知识库"按钮
2. 应该看到6个技巧分类卡片
3. 点击"刀工基础"卡片
4. 应该弹出详情模态框
5. 查看课程列表（应该有8个课程）
6. 点击第一个课程"持刀姿势与安全"
7. 应该显示详细的课程内容
8. 点击其他课程，内容应该切换
9. 点击右上角X关闭模态框
10. 测试其他技巧分类

**成功标志**:
- 所有6个分类都能打开
- 课程数量正确（刀工8个、火候6个等）
- 课程内容详细且格式正确
- 模态框可以正常关闭

##### ✅ 实时指导模式
1. 点击"实时指导"按钮
2. 应该看到"选择一个菜谱开始实时指导"
3. 点击"去生成菜谱"按钮
4. 在主页生成一个菜谱
5. 返回"去学厨房" > "实时指导"
6. 应该显示菜谱步骤
7. 点击"下一步"按钮
8. 步骤应该切换
9. 查看AI提示是否显示
10. 测试"上一步"按钮

**成功标志**:
- 能够选择菜谱
- 步骤导航正常
- AI提示显示
- 按钮状态正确（第一步时"上一步"禁用）

##### ✅ 问题诊断模式
1. 点击"问题诊断"按钮
2. 点击上传区域
3. 选择一张图片
4. 图片应该显示预览
5. 输入问题："炒出来的菜太咸了"
6. 点击"开始诊断"
7. 等待诊断结果（应该看到"诊断中..."）
8. 查看诊断结果是否详细

**成功标志**:
- 图片上传成功
- 诊断结果详细（包含原因、解决方案、预防建议）
- 不是备用诊断

## 常见问题排查

### 问题1: AI总是返回备用答案

**症状**:
```
这是一个很好的问题！建议您：
1. 多观察食材的变化
2. 掌握基本的烹饪原理
3. 多实践多总结
4. 可以参考专业烹饪书籍或视频教程
```

**原因**:
- 豆包API调用失败
- API Key无效
- 网络问题

**解决方案**:
1. 检查 `backend/.env` 文件
2. 验证API Key是否有效
3. 运行 `python test_cooking_ai.py` 查看详细错误
4. 检查网络连接到豆包API

### 问题2: 技巧知识库点击无反应

**症状**:
点击技巧卡片没有弹出模态框

**原因**:
- SkillDetailModal组件未正确导入
- 事件处理函数未绑定

**解决方案**:
已修复，确保使用最新代码

### 问题3: 前端无法连接后端

**症状**:
```
AI服务暂时不可用，请稍后重试
```

**原因**:
- 后端未启动
- 端口冲突
- CORS配置问题

**解决方案**:
1. 确认后端正在运行（应该在 http://localhost:8000）
2. 检查后端日志是否有错误
3. 确认前端API配置正确

### 问题4: 图片上传失败

**症状**:
上传图片后没有反应或报错

**原因**:
- 图片格式不支持
- 图片太大
- 后端uploads目录不存在

**解决方案**:
1. 使用JPG/PNG格式
2. 确保图片小于10MB
3. 检查 `backend/uploads` 目录是否存在
