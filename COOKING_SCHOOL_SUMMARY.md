## 关于AI功能的说明

### AI调用流程
1. 用户提交问题
2. 前端调用API（`/api/cooking/ask` 或 `/api/cooking/diagnose`）
3. 后端构建专业的提示词
4. 调用豆包API（最多重试3次）
5. 解析响应并返回
6. 如果API失败，返回备用答案

### 为什么会返回备用答案？
如果你测试时发现AI总是返回类似"这是一个很好的问题！建议您：1. 多观察食材的变化..."这样的回答，可能是因为：

1. **豆包API调用失败** - 这是最常见的原因
2. **API Key无效或过期**
3. **网络连接问题**
4. **API端点URL错误**

### 如何检查和解决？

#### 1. 查看后端日志
启动后端时，日志会显示详细的API调用信息：
```
INFO: 调用豆包API回答烹饪问题 (尝试 1/3)
DEBUG: API URL: https://ark.cn-beijing.volces.com/api/v3/chat/completions
DEBUG: API Key (前8位): 8b12a07e...
DEBUG: Model: doubao-seed-1-8-251228
INFO: API响应状态码: 200
```

#### 2. 运行测试脚本
```bash
cd backend
python test_cooking_ai.py
```

这个脚本会：
- 显示API配置信息
- 测试多个问题
- 显示详细的响应
- 标记成功/失败

#### 3. 检查配置文件
确认 `backend/.env` 文件中的配置：
```env
LLM_API_KEY=8b12a07e-b47a-4e39-8e36-c96c7ccced86
LLM_API_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
MODEL_NAME=doubao-seed-1-8-251228
```

#### 4. 验证API Key
- 登录豆包API控制台
- 检查API Key是否有效
- 确认是否有足够的配额
- 验证模型名称是否正确
